RESOURCES-FORMAT

RESOURCES.md 的格式规范

RESOURCES.md is the curated set of trusted sources for this topic. Knowledge for explainers should be drawn from here, not from parametric guesses. Wisdom comes from the communities listed here.

RESOURCES.md 是该主题的精选可信资源集。讲解用的知识应从这里提取，而非依赖参数化猜测。智慧来自此处列出的社区。

Structure

结构

Rules

规则

  • High-trust only. Prefer primary sources, recognised experts, peer-reviewed work, and communities with strong moderation. If a resource is marketing dressed as education, leave it out.
  • Annotate every entry. A bare link is useless in three months. Add one line: what it covers and when to reach for it.
  • Group by Knowledge / Wisdom. Mirrors the philosophy in SKILL.md. It is fine for a resource to appear in only one group.
  • Surface gaps explicitly. If no good resource exists for an area the mission needs, write a ## Gaps section listing what is missing. This drives future search.
  • Prune ruthlessly. A resource that turned out to be wrong, shallow, or off-mission should be removed, not buried. Better five sharp sources than thirty mediocre ones.
  • Record community preferences. If the user has opted out of joining communities, note it here so future sessions don't keep proposing them.
  • 仅限高可信度。优先选择一手资源、公认专家、同行评审的工作和管理良好的社区。如果一个资源是伪装成教育的营销内容，就不要收录。
  • 为每个条目添加注释。一个孤零零的链接三个月后就毫无用处了。加一行说明：它涵盖什么内容，何时该使用它。
  • 按知识 / 智慧分组。对应 SKILL.md 中的理念。一个资源只出现在一个分组中也完全可以。
  • 明确标出空白。如果使命所需的某个领域没有好的资源，写一个 ## Gaps 部分列出缺失的内容。这能驱动后续的搜索。
  • 果断修剪。被证明是错误、浅薄或偏离使命的资源应该被删除，而非埋没。五个精挑细选的资源胜过三十个平庸的来源。
  • 记录社区偏好。如果用户选择不加入社区，在此记录，以免未来的会话反复提议。
